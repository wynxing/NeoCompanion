## 2. Problem Statement

### 2.1 Market Gap

当前 AI 产品主要集中在三类能力：
1. 问答与生成
2. 效率与任务处理
3. 陪伴与角色互动

这些产品存在以下局限性：
- **纯问答型 AI**：缺乏持续陪伴状态，用户需要反复手动复制上下文，难以融入脑力活动的真实过程。
- **效率管理工具**：冰冷机械，缺乏温度，容易沦为“任务记录本”，甚至带给用户被监控的心理压力。
- **纯陪伴类产品**：偏重情感和娱乐聊天，难以参与实质性的学习、编程、创作等复杂脑力行为。
- **自主型 Agent**：过度追求全自动代替用户做决策，面临极高的出错率和隐私侵入性。

### 2.2 User Problem

目标用户（如学生、开发者、创作者等脑力活动者）的痛点在于：
- **难以开始**：面对复杂学习或工作任务时容易拖延，缺乏仪式感和即时正面反馈。
- **难以持续**：专注时容易被打断、分心或在多窗口切换中迷失。
- **卡壳与焦虑**：遇到高难度问题思路中断时，没有懂你的伙伴给予引导或舒缓情绪。
- **连接性欠缺**：自己写的辅助脚本、编译结果、监控状态等，无法以自然、低门槛的方式转化为日常感知。

### 2.3 Why This Matters

NeoCompanion 致力于通过**「桌宠情感载体 + 屏幕状态感知 + 智能网关连接 + 开放 Hook API」**的融合架构，解决这一痛点，让 AI 从“一次性应答工具”升级为“懂状态、能联动、长久相伴的数字搭子”。

## 3. Product Definition

### 3.1 Positioning

NeoCompanion 是一款**融合了「桌面悬浮虚拟宠物 + 陪伴式监督学习 + 智能连接网关」的长期陪伴型 AI 系统**。

以可爱、有温度的悬浮桌宠（Live2D / 动态帧动画）为前端载体，通过屏幕状态感知的温情反馈，辅以简单日常记录与智能网关协议复用，帮助用户在学习和创作中克服拖延、卡壳与中断。

### 3.2 Core Product Decisions

本文档锁定以下产品核心判断：
1. **产品形态与主轴**：以桌面悬浮桌宠及拟人化 TTS 语音互动为核心界面，主打“有温度的陪伴监督学习”，绝非机械式效率看板。
2. **状态感知温情化**：屏幕识别唯一目的是为了“感知用户当前的专注/分心/卡壳/疲劳状态”，并由宠物给予匹配的情绪反馈（如递虚拟咖啡、催促休息），严禁做任何商业监控。
3. **网关架构与大模型对话**：
   - 不从零做庞大笨重的 Autonomous Agent。
   - NeoCompanion 作为**连接网关**与**对话适配器**。
   - 复杂跨软件任务代行调起 [OpenClaw](https://github.com/openclaw/openclaw)；日常答疑与编程辅助直接调用标准大模型 API，融入桌宠极简对话流。
4. **极客定制与 Hook 连接**：核心功能包含开放的 Hook API 监听接口，支持用户用自己编写的自定义程序推送编译、监控等状态提醒，桌宠进行拟人化播报。

## 4. Target Users

### 4.1 Core User

- **学生群体**：用于长期学业规划、番茄钟陪同专注、背词与论文写作卡壳引导，对正面情感反馈敏感。
- **开发者与极客群体**：用于长期的深夜写代码陪伴、代码错误通过 Claude 调试面板适配、构建与服务器监控 Hook 拟人化提醒。
- **创作者群体**：用于灵感随手记、日常琐碎待办打理、写作进度监督。

## 5. Capability Model

NeoCompanion 的长期能力分为四层：

```
+-------------------------------------------------------+
|  1. Companion Layer (桌宠悬浮交互 / TTS 语音 / 情绪反馈)  |
+-------------------------------------------------------+
|  2. Routine Layer   (番茄钟 / Todo账本 / 天气 / 宠物小日记) |
+-------------------------------------------------------+
|  3. Gateway Layer   (内置原子Agent / OpenClaw / LLM 极简对话) |
+-------------------------------------------------------+
|  4. Hook & System   (本地安全Hook API / 本地状态感知与记忆) |
+-------------------------------------------------------+
```

### 5.1 Companion Layer (陪伴交互层)
- **Live2D / 动态帧动画**：桌面常驻，支持拖拽、置顶、多种宠物动作响应。
- **TTS 语音合成**：拟人化声线和语气交互，告别冷冰冰的纯文字提示。
- **状态化情绪反馈**：配合用户状态触发嘟嘴、欢呼、看书、递虚拟咖啡等动作。

### 5.2 Routine Layer (日常琐碎陪伴层)
- **陪伴式番茄钟**：桌宠与用户同步进入专注计时，陪伴小动作。
- **宠物待办小账本**：用账本包装 Todo，拟人化语气催促和打气。
- **日常天气与碎碎念**：基于气象的保暖带伞提醒、早晚问候、健康作息劝导。
- **宠物视角小日记**：日终自动生成以宠物口吻撰写的温度成长记录。

### 5.3 Gateway Layer (智能网关与大模型对话层)
- **内置基础 Agent**：微型文件读写、网页搜索总结、剪贴板获取。
- **OpenClaw 任务代行网关**：把繁复、长流程的跨软件操作分发给 [OpenClaw](https://github.com/openclaw/openclaw)，NeoCompanion 面板进行友好状态展示。
- **轻量 AI 对话与开发辅助**：支持通过聊天面板直连大模型（如 DeepSeek/Claude）进行日常提问、思路启发或一键拾取剪贴板报错，提供代码调优建议，保持极简免打扰。

### 5.4 Hook & System Layer (系统与 Hook 层)
- **安全本地多通道 Hook**：暴露本地端口监听，允许用户自定义脚本/程序向桌宠推送状态。同时支持自动检测主流 AI 编程终端及 Agent 配置文件进行自动打桩注入。
- **浮动权限审查气泡**：支持当外部 Agent 执行敏感的写操作或命令时，通过桌宠弹出半透明审批气泡 (Permission Bubble)，允许用户通过快捷热键一键审批，实现终端与桌面的温情交互。
- **隐私感知引擎**：本地运行窗口特征提取，去除敏感及隐私正文，仅用最小必要维度去推算脑力状态。
- **本地长期记忆**：记录用户专注偏好、被提醒习惯，使宠物在陪伴过程中更加默契。

## 6. Roadmap Framework

### 6.1 v1: 陪伴桌宠最小闭环 (Warm Companion Base)
- **目标**：实现桌面常驻悬浮宠物的基本 Live2D 形态、语音 TTS 播报。
- **核心功能**：宠物陪伴番茄钟、宠物待办、天气碎碎念。
- **感知反馈**：检测活跃窗口类型，对专注与严重分心进行宠物发声催促。

### 6.2 v2: 开放 Hook 极客生态与 UDS 本地挂载 (Custom Hook, UDS & AI Chat)
- **目标**：实现极客扩展、零端口开销与轻量陪伴能力。
- **核心功能**：
  - 发布安全本地 Hook API，支持网络与**免代码本地文件监听哨兵 (File Watcher)** 多通道状态推送至桌宠提醒。
  - **实现零端口本地域套接字 (UDS) / Named Pipe 本地挂载支持**，杜绝本地 TCP 端口占用冲突。
  - 实现自动扫描并注入 OpenClaw 等工具的插件钩子配置。
  - 落地“浮动权限审批气泡”浮窗卡片，绑定全局快捷键 (`Ctrl+Shift+Y` / `Ctrl+Shift+N`) 快捷流审批。
  - 完善本地 AI 对话的多模型自定义配置，提供一键拾取剪贴板报错和高效答疑的极速体验。

### 6.3 v3: 完整智能网关与分布式远程挂载生态 (Full AI Companion & Distributed Hosting OS)
- **目标**：作为个人工作流的统一网关伙伴系统，支持主机算力解耦。
- **核心功能**：
  - **支持分布式远程宿主挂载 (Remote Server Binding)**，允许将 LanceDB 向量检索、Drizzle SQLite 等重度模块托管至 NAS 或云服务器，极大卸载本地工作机算力并支持多设备数据同步。
  - 深度接入 OpenClaw，在桌宠侧适配展示复杂跨应用操作进度。
  - 宠物性格记忆演进，形成有长期偏好和高度默契的情感交互。
