## 1. 产品定位与核心形态 (Product Definition & Form Factor)

NeoCompanion 是一款**融合了「桌面智能助手 + 陪伴式监督学习 + 智能连接网关」的长期陪伴型 AI 系统**。

我们希望打破传统 AI 工具“打开-提问-拿结果-关闭”的冰冷交互模式，让 AI 以一个可爱、有温度的**桌面悬浮助手**形态长期存在于用户的学习、工作和创作现场，陪同用户共同生活与成长。

```
+-------------------------------------------------------+
|  Windows / macOS 桌面                                  |
|                                                       |
|   [ 用户正在写代码/写论文 ]                             |
|                                                       |
|         ( ^_^ )  <-- 桌面悬浮助手 (2D 精灵图帧动画)  |
|        /  o   o \    语音 TTS 反馈: "加油哦，你已经专注20分钟啦！"|
|                                                       |
+-------------------------------------------------------+
```

### 1.1 核心形态特征
- **悬浮助手形态**：轻量级桌面悬浮窗，支持 2D 精灵图 或动态帧动画，支持用户拖拽、置顶、缩放、隐藏至边缘。
- **壁纸层状态形态**：助手的状态信息（天气时间、任务清单、专注计时、助手寄语）以极淡、半透明的方式融入桌面壁纸层。壁纸上的元素像壁纸本身一样"安静"——你看到它，但你不会想去"点"壁纸。所有交互仍通过悬浮助手或面板完成。
- **语音 TTS 反馈**：助手拥有拟人化的个性声线，能够用自然的语气和语调进行日常互动、提醒、吐槽和安慰，让陪伴从“读文字”升华为“真对话”。
- **监督学习仪式感**：
  - **任务开始**：助手会做伸懒腰或加油鼓劲的动作，播放鼓励语音，为用户建立低门槛的“启动仪式”。
  - **任务中途**：低打扰陪伴，偶有呼吸或看书等小动作。
  - **分心/偏离**：当感知到用户偏离任务时，助手会做出轻轻叹气或小声嘟囔的催促互动。
  - **任务完成**：助手做出庆祝动作，给予正面反馈，提升用户的成就感。

## 2. 日常琐碎陪伴与记录能力 (Routine Companion & Logging)

助手不仅是观赏性的花瓶，更是协助管理日常学习与生活的微型工作台。这些日常功能通过助手的口吻与动作进行助手风格包装。

### 2.1 助手番茄钟
- **共同专注**：开启番茄钟后，悬浮助手会进入”学习/工作状态”（例如戴上眼镜、翻开小书本或同步敲击迷你小键盘）。
- **壁纸专注计时环**：专注期间，桌面壁纸右下角出现极淡的 SVG 圆环计时器，显示剩余分钟数。切回桌面时无需任何操作即可感知进度。计时环只做展示，开始/暂停仍通过助手右键菜单或面板操作。
- **打扰管理**：在番茄钟专注期间，助手会帮用户抵挡非紧急弹窗，并用简短语音提醒：“专注中，保持节奏。”
- **互动休息**：番茄时间结束，助手会主动伸懒腰，用温和地提醒用户：“专注时间到了，建议休息一下。”

### 2.2 拟人化待办助手 (Todo Assistant)
- **助手任务清单**：待办事项列表被包装为”助手任务清单”。
- **壁纸任务提醒**：桌面壁纸左侧竖向显示未完成任务清单（最多 5 项），极淡透明度。回到桌面时一瞥便知还有什么没做。但壁纸上不提供勾选——勾选的满足感留给面板，完成需要仪式感。
- **轻量提醒**：不会以生硬的系统弹窗提醒，而是由助手抱出一个小看板，或者用语音轻声提醒：“你今天还有‘写论文大纲’的承诺没兑现哦，我们要不要现在开始？”
- **动态进度**：当用户划掉待办时，助手会做出拍手或开心的动作。

### 2.3 天气提醒与日常碎碎念
- **智能气象陪伴**：连接本地天气 API，在清晨启动、即将下雨或气温剧变时，助手会主动提醒：”外面在下雨，出门记得带伞。”
- **壁纸天气时间**：桌面壁纸顶部常驻显示天气图标+温度+时间日期，极细等宽字体，低透明度。这是壁纸层最”天然”的元素——用户已习惯在壁纸上看到时钟，天气与之并列提供环境感知。
- **时间感应碎碎念**：
  - 深夜：“夜深啦，小星星都睡了，我们也准备回被窝吧？”
  - 下午：“下午容易犯困，要不要喝杯茶，我给你放首轻音乐？”

### 2.4 助手视角“个人小日记”记录
- **温情记录**：助手会记录用户一天的专注时长、完成的任务以及克服卡壳的瞬间，在一天结束时生成一篇“以助手视角撰写的小日记”发送给用户。
- **成长轨迹**：日记充满温度，而非冷冰冰 of 折线图，帮助用户在长期的脑力活动中建立积极的情感回馈。

### 2.5 壁纸层氛围感知

助手不仅能"说话"和"做动作"，还能让整个桌面"呼吸"。

- **氛围色调**：壁纸层叠加一个极淡（近乎不可见）的全屏渐变色调层。专注时微微暖光、分心时极淡微红提醒、深夜时偏暗蓝。色调变化用 3 秒缓慢过渡，用户甚至不会意识到壁纸颜色在变，但会直觉地感到"氛围"不同了。
- **助手寄语**：状态切换时，壁纸中央偏下浮现一句极淡的反馈文字（如"保持专注，你正在做重要的事"），10 秒后自动淡出。寄语不是给"看"的，是给"感受"的——偶尔余光扫到，会感受到陪伴的存在。
- **环境融合**：壁纸层所有元素都遵循"隐入环境"原则——低透明度、极细字体、无交互区域。信息像壁纸的一部分在"呼吸"，而非贴上去的卡片。

## 3. 温情化的屏幕感知与个性化陪伴 (State Perception via Screen Recognition)

为了让助手更加“懂你”，NeoCompanion 具备轻量的**屏幕识别与状态感知能力**。屏幕识别的唯一出发点是**观察用户状态，以生成更具个性的长久陪伴反馈**，而非企业级的隐私监控。

```
+------------------+     本地最小特征提取     +------------------+
| 用户屏幕/窗口状态  | ---------------------> | 助手状态决策引擎  |
+------------------+                          +------------------+
                                                       |
                                                       v 动态驱动表情/语音
                                              +------------------+
                                              | “哎呀，怎么卡在这里啦，|
                                              |  要不要换个思路呢？”  |
                                              +------------------+
```

### 3.1 状态判断逻辑
系统通过本地极轻量级的窗口标题、进程名或授权下的局部屏幕内容特征提取，分析用户当前的脑力状态：
- **卡壳状态 (Stuck)**：用户在同一个代码文件、同一个文档编辑页面停留时间过长，打字频率骤降或完全停止。
- **分心状态 (Distracted)**：在非休息时间，频繁且长时间地切换到购物网站、娱乐视频或游戏进程。
- **疲劳状态 (Exhausted)**：深夜高强度写代码或写文档已持续数小时，或连续专注时间过长。
- **高效状态 (Flow)**：打字流畅、应用切换高度聚焦于开发与参考资料之间。

### 3.2 个性化温情反馈
感知到状态后，悬浮助手会产生与当前状态高度匹配 of 拟人化反馈：
- **遇到卡壳**：助手会歪着脑袋，递上一杯“虚拟热咖啡”，语音提示：“看起来思路卡住了？要不要换个角度试试，或者帮你搜一下相关资料？”
- **偏离分心**：助手会轻轻敲敲地敲敲屏幕屏幕边缘，温和地提醒：“看起来分心了，要继续专注吗？”
- **深夜疲劳**：助手会打哈欠，揉揉眼睛，温和地建议：“已经工作三个小时了，建议休息一下。”

### 3.3 隐私安全红线
- **本地优先**：所有屏幕和行为数据完全在本地计算和消耗，绝不上传云端，保障绝对的隐私安全。
- **脱敏提取**：仅提取非敏感的特征结构（例如“停留时长”、“应用类型属性”），屏蔽一切密码、个人身份及敏感聊天内容。

## 4. 本地知识工作空间与 AI 对话 (Local Knowledge Workspace & AI)

NeoCompanion 不把复杂跨应用自动执行作为产品主轴，而是围绕用户主动维护的知识与行动建立闭环：笔记负责沉淀信息，任务和看板负责推进工作，AI 通过检索引用这些内容提供帮助。

```
                +-----------------------------+
                |       NeoCompanion          |
                | 项目 / 笔记 / 看板 / AI 对话  |
                +--------------+--------------+
                               |
                 本地全文检索 + 可选向量检索
                               |
                +--------------v--------------+
                | SQLite: 业务数据 / FTS5 / vec |
                +--------------+--------------+
                               |
                    用户授权后调用外部 API
                               |
                +--------------v--------------+
                | Chat Model / Embedding API   |
                +-----------------------------+
```

### 4.1 工作空间与项目
- **单一本地工作空间**：首版不引入账号、团队空间或云同步，所有内容保存在当前设备。
- **项目作为组织边界**：笔记、看板和任务均可归属项目；系统提供不可缺省的“收件箱”项目承接未分类内容。
- **统一任务系统**：简单待办与看板卡片是同一份任务数据的不同视图，避免重复录入和状态不一致。

### 4.2 Markdown 笔记与看板
- **Markdown 笔记**：首版提供标题、正文、标签和预览，不实现 Notion 式块编辑、附件、协作和版本历史。
- **基础看板**：项目可包含看板与有序列；任务支持跨列移动、排序和完成状态。
- **内容关联**：AI 检索覆盖笔记和任务，来源可以直接打开对应内容，形成“资料 → 行动 → 回顾”的连续工作流。

### 4.3 混合检索与索引生命周期
- **全文检索**：SQLite FTS5 索引标题、Markdown 正文、标签和任务描述，保证专有名词和精确关键词可命中。
- **向量检索**：`sqlite-vec` 保存语义向量；Markdown 按标题和段落稳定分块，单块约 1200 字符并保留少量重叠，任务作为单独知识条目。
- **结果融合**：全文和向量候选通过排名融合产生最终结果，并严格遵守当前项目或全部项目的检索范围。
- **增量更新**：内容保存后异步生成索引；删除内容同步清除全文和向量记录；Embedding 模型或维度变化时将旧索引标记为待重建。
- **明确降级**：Embedding 未配置、额度不足、网络失败或向量扩展不可用时，系统继续提供全文搜索，并在界面显示索引状态。

### 4.4 有来源的 AI 对话
- **知识增强问答**：AI 默认检索当前项目，用户可切换为全部项目；检索结果作为上下文发送给聊天模型。
- **来源约束**：回答后的来源列表只能包含本次实际召回的笔记或任务，不允许模型虚构引用。
- **隐私边界**：业务数据、全文索引和向量索引默认留在本地。只有用户配置并启用 Provider 后，待向量化文本和问答上下文才会发送给对应云端服务。
- **模型解耦**：聊天模型与 Embedding Provider 分开配置，首版通过 OpenAI-compatible `/embeddings` 接口接入，不假定聊天模型具备向量能力。

## 5. 可自定义 Hook 状态跟踪与扩展机制 (Custom Hooks & Reminders)

为了给开发者和极客用户提供极致的自定义空间，NeoCompanion 提供了强大的 **Hook 连接机制**。

日常使用中，用户可以把 NeoCompanion 作为统一的**数字状态看板与拟人化提醒中心**，通过 Hook 连接任何第三方自定义程序。

```
+--------------------+                 Hook 推送 (HTTP/WS)
| 用户自定义程序/脚本  | ------------------------------------+
| (如编译脚本/监控脚本) |                                     |
+--------------------+                                     v
                                                  +-----------------+
                                                  |  NeoCompanion   |
                                                  |  助手拟人化提醒   |
                                                  +-----------------+
                                                  | "报告！ |
                                                  | 编译通过啦，真棒！"|
                                                  +-----------------+
```

### 5.1 本地 Hook 接口设计
- **简易 HTTP API / WebSocket 接收端**：程序在本地监听一个安全端口（例如 `localhost:10103`），提供标准的 JSON 接口，接收外部推送。
- **自定义状态定义**：外部推送 of JSON 结构可包含：
  - `status_type`: 状态类型（如：成功、警告、严重错误、日常通知）。
  - `message`: 具体消息文本。
  - `pet_action`: 期望助手做出的指定动作/表情编号。
  - `tts_voice`: 是否开启语音反馈，以及指定的语音文案。

### 5.2 典型自定义 Hook 实例
1. **开发者编译与部署通知**：
   - 编写一段 `git push` 或代码构建后的 Hook 脚本。
   - 一旦编译成功，脚本 Hook 推送给助手。助手在屏幕上跳舞并用语音欢呼：“部署成功，没有报错。”
   - 一旦构建失败，助手则做出提醒动作的表情，语音提示：“编译失败了，需要看看报错吗？”
2. **服务器与服务监控提醒**：
   - 监控本地或远程服务的健康状态。一旦服务挂掉，自动 Hook 触达助手，助手做出紧张的动作发出语音警报。
3. **第三方智能家居与健康跟踪**：
   - 比如番茄钟久坐、饮水机提醒、社交媒体收到新私信，皆可通过轻量 Hook 直接让悬浮助手为您播报。

### 5.3 多通道 Hook 联动架构 (Advanced Multi-channel Hooks)

Hook 是独立的外部状态入口，不绑定任何特定 Agent 或 CLI。外部程序可主动选择 HTTP、WebSocket、文件监听或本地域套接字接入，NeoCompanion 不扫描或改写第三方工具配置。

#### 5.3.1 多物理通道主服务挂载 (Local Transport Modes)
系统不局限于单一的 `localhost:PORT` 绑定，提供本机通信方式以适应不同环境：
- **UDS / Named Pipe 挂载（高安零端口）**：在对安全性要求极高的环境，或者由于端口占用导致频繁报错时，系统支持挂载到本地 Unix Domain Socket (`/tmp/neo-companion.sock`) 或 Windows Named Pipe (`\\.\pipe\neo-companion`)。**100% 避免端口冲突与防火墙弹窗警告**。

#### 5.3.2 浮动权限卡片与快捷审批 (Permission Bubble & Hotkeys)
- **免死等终端**：当已接入的外部程序请求执行敏感指令（如修改核心系统配置、执行高危脚本、覆盖关键文件）时，可通过通用 Hook API 挂起流程并向 NeoCompanion 发送审批请求。
- **透明审批浮窗**：助手会直接在桌面弹出一个精美、半透明的”权限气泡审批卡片 (Permission Bubble)”，展示操作详情（操作类型、涉及路径、敏感度评分）。
- **极速热键反馈**：用户可以直接点击审批卡片中的 `Allow (同意)` 或 `Deny (拒绝)`。极客用户可以使用系统级全局热键：`Ctrl+Shift+Y` 批准，`Ctrl+Shift+N` 拒绝。
- **状态流控**：审批结果通过 HTTP/WebSocket 异步返回给挂起的执行进程，让终端交互实现从命令行到桌面有温度交互的跃升。

#### 5.3.3 角标通知提醒 (Badge Notification)
当外部 Agent 推送状态通知或审批请求时，助手**不会**在壁纸上弹出通知条（壁纸层不交互），而是在悬浮助手的右上角显示一个**红色角标**：
- 1 条未读：红色圆点
- 多条未读：红色圆点 + 数字
- 无未读：角标隐藏

角标的设计隐喻来自手机 App 的未读提示——用户已有”红点=有待处理”的直觉认知。点击角标后打开面板，切换到 Hook 确认视图进行操作。角标本身不承载任何操作，只做”轻量提醒”。

#### 5.3.4 免代码文件监听哨兵 Hook (Zero-Code File Watcher Hook)
为了让非开发人员或编写简单 Bash 脚本的极客也能轻松“挂载”状态，系统提供**文件监听哨兵机制**：
- 挂载后台监听 `~/.neo-companion/hooks/` 目录。
- 任何脚本在执行成功、失败或完成某项任务时，只需以标准 Shell 重定向指令写入 JSON 即可：
  `echo '{"status": "success", "message": "代码同步已完成！", "tts": "代码同步完成。"}' > ~/.neo-companion/hooks/sync.json`
- 系统将秒级感知该文件覆写，自动触发助手的欢呼动作与 TTS 播报。**完全避免了在受限命令行环境里处理 curl 网络请求超时或失败的困扰**。

#### 5.3.5 局域网消息总线挂载 (MQTT / Home Assistant Hook)
支持将助手服务挂载并订阅本地家庭局域网的 MQTT 消息队列（如 `home/sensors/sitting`），使得智能家居状态（如久坐报警、饮水提醒、编译网关状态）可以实时无缝驱动助手在桌面做出温馨催促。

#### 5.3.6 微观状态映射引擎 (Event-to-Animation Mapping)
助手通过 Hook 捕获到外部 Agent 的微小执行状态时，能完美映射到助手的细腻动画和动作中：
- **Thinking (思考中)**：Agent 正在梳理规划、调用模型时，助手做出托腮、思考、或转笔动作。
- **Working/Typing (工具执行中)**：Agent 正在跑工具、修改文件时，助手戴上眼镜在迷你键盘上工作。
- **Error (发生错误)**：任务失败或编译出错时，助手抱头哭泣或做出生气动作，并伴有心疼的语音安慰。
- **Subagent Juggling (多子智能体协作)**：当 Agent 启动多进程、多子智能体并行协作时，助手开始耍三个杂耍球，象征高效运转。

## 6. 阶段演进路线 (Roadmap)

### 6.1 Phase 1: 温暖的助手 (Warm Desk-Pet Base)
- **目标**：实现桌面悬浮助手的 2D 精灵图 动态展示、可爱的语音 TTS 反馈。
- **功能**：落地助手番茄钟、助手 Todo 小账本、日常天气播报和记录日记。
- **感知**：完成基本的窗口活跃度检测，驱动助手对“专注/分心”做出语音和表情互动。

### 6.2 Phase 2: 本地知识工作空间 (Local Knowledge Workspace)
- **目标**：完成“记录 → 组织 → 检索 → 引用回答”的知识闭环。
- **功能**：
  - 项目、Markdown 笔记、统一任务与基础看板。
  - SQLite FTS5 与 `sqlite-vec` 混合检索、增量索引、索引状态和重建。
  - 聊天模型与 Embedding Provider 独立配置，AI 回答展示可点击来源。
  - 保留通用 Hook API、文件监听、UDS / Named Pipe 与权限审批能力，但不自动修改第三方配置。
- **验证**：在无向量 Provider 时可完成笔记与看板管理和全文搜索；配置 Provider 后可得到带真实来源的知识问答。

### 6.3 Phase 3: 知识扩展与长期陪伴 (Knowledge Expansion & Long-term Companion)
- **目标**：扩展知识来源和个性化能力，同时保持本地优先与用户可控。
- **功能**：
  - 支持指定 Markdown 文件夹同步，以及 PDF、网页等多格式导入。
  - 提供可选本地 Embedding 模型，减少知识正文发送到云端的需要。
  - 将用户知识和助手长期记忆分区治理，支持查看来源、独立开关与删除。
  - 在显式授权下扩展偏好记忆，使助手逐渐形成可解释、可撤销的个性化策略。
