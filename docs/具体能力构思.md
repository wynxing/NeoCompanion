## 1. 产品定位与核心形态 (Product Definition & Form Factor)

NeoCompanion 是一款**融合了「桌面虚拟宠物 + 陪伴式监督学习 + 智能连接网关」的长期陪伴型 AI 系统**。

我们希望打破传统 AI 工具“打开-提问-拿结果-关闭”的冰冷交互模式，让 AI 以一个可爱、有温度的**桌面悬浮宠物**形态长期存在于用户的学习、工作和创作现场，陪同用户共同生活与成长。

```
+-------------------------------------------------------+
|  Windows / macOS 桌面                                  |
|                                                       |
|   [ 用户正在写代码/写论文 ]                             |
|                                                       |
|         ( ^_^ )  <-- 桌面悬浮宠物 (Live2D / 动态帧动画)  |
|        /  o   o \    语音 TTS 反馈: "加油哦，你已经专注20分钟啦！"|
|                                                       |
+-------------------------------------------------------+
```

### 1.1 核心形态特征
- **悬浮桌宠形态**：轻量级桌面悬浮窗，支持 Live2D 或动态帧动画，支持用户拖拽、置顶、缩放、隐藏至边缘。
- **语音 TTS 反馈**：宠物拥有拟人化的个性声线，能够用自然的语气和语调进行日常互动、提醒、吐槽和安慰，让陪伴从“读文字”升华为“真对话”。
- **监督学习仪式感**：
  - **任务开始**：宠物会做伸懒腰或加油鼓劲的动作，播放鼓励语音，为用户建立低门槛的“启动仪式”。
  - **任务中途**：低打扰陪伴，偶有呼吸或看书等小动作。
  - **分心/偏离**：当感知到用户偏离任务时，宠物会做出轻轻叹气或小声嘟囔的催促互动。
  - **任务完成**：宠物欢呼雀跃，给予正面反馈，提升用户的成就感。

## 2. 日常琐碎陪伴与记录能力 (Routine Companion & Logging)

宠物不仅是观赏性的花瓶，更是协助管理日常学习与生活的微型工作台。这些日常功能通过宠物的口吻与动作进行拟人化包装。

### 2.1 宠物专属番茄钟
- **共同专注**：开启番茄钟后，悬浮宠物会进入“学习/工作状态”（例如戴上眼镜、翻开小书本或同步敲击迷你小键盘）。
- **打扰管理**：在番茄钟专注期间，宠物会帮用户抵挡非紧急弹窗，并用简短语音提醒：“嘘，我们在闭关修炼哦。”
- **互动休息**：番茄时间结束，宠物会主动伸懒腰，用欢快的语气催促用户：“时间到啦！快喝口水，跟我一起揉揉眼睛。”

### 2.2 拟人化待办助手 (Todo Assistant)
- **宠物小账本**：待办事项列表被包装为“宠物的任务小账本”。
- **轻量提醒**：不会以生硬的系统弹窗提醒，而是由宠物抱出一个小看板，或者用语音轻声提醒：“主子，你今天还有‘写论文大纲’的承诺没兑现哦，我们要不要现在开始？”
- **动态进度**：当用户划掉待办时，宠物会做出拍手或开心的动作。

### 2.3 天气提醒与日常碎碎念
- **智能气象陪伴**：连接本地天气 API，在清晨启动、即将下雨或气温剧变时，宠物会主动发声：“主子，外面好像在下雨呢，出门记得带伞哦，别感冒了！”
- **时间感应碎碎念**：
  - 深夜：“夜深啦，小星星都睡了，我们也准备回被窝吧？”
  - 下午：“下午容易犯困，要不要喝杯茶，我给你放首轻音乐？”

### 2.4 宠物视角“个人小日记”记录
- **温情记录**：宠物会记录用户一天的专注时长、完成的任务以及克服卡壳的瞬间，在一天结束时生成一篇“以宠物视角撰写的小日记”发送给用户。
- **成长轨迹**：日记充满温度，而非冷冰冰 of 折线图，帮助用户在长期的脑力活动中建立积极的情感回馈。

## 3. 温情化的屏幕感知与个性化陪伴 (State Perception via Screen Recognition)

为了让宠物更加“懂你”，NeoCompanion 具备轻量的**屏幕识别与状态感知能力**。屏幕识别的唯一出发点是**观察用户状态，以生成更具个性的长久陪伴反馈**，而非企业级的隐私监控。

```
+------------------+     本地最小特征提取     +------------------+
| 用户屏幕/窗口状态  | ---------------------> | 宠物状态决策引擎  |
+------------------+                          +------------------+
                                                       |
                                                       v 动态驱动表情/语音
                                              +------------------+
                                              | “哎呀，怎么卡在这里啦，|
                                              |  要不要换个思路呢？”  |
                                              +------------------+
```

### 3.1 状态判断逻辑
系统通过本地极轻量级的窗口标题、进程名或授权下的局部屏幕内容特征提取，分析用户当前的脑力状态：
- **卡壳状态 (Stuck)**：用户在同一个代码文件、同一个文档编辑页面停留时间过长，打字频率骤降或完全停止。
- **分心状态 (Distracted)**：在非休息时间，频繁且长时间地切换到购物网站、娱乐视频或游戏进程。
- **疲劳状态 (Exhausted)**：深夜高强度写代码或写文档已持续数小时，或连续专注时间过长。
- **高效状态 (Flow)**：打字流畅、应用切换高度聚焦于开发与参考资料之间。

### 3.2 个性化温情反馈
感知到状态后，悬浮宠物会产生与当前状态高度匹配 of 拟人化反馈：
- **遇到卡壳**：宠物会歪着脑袋，递上一杯“虚拟热咖啡”，语音提示：“主子，是不是思路卡住了？要不要听我讲个冷笑话放松一下，或者咱们去搜索一下相关的资料？”
- **偏离分心**：宠物会气鼓鼓地敲敲屏幕屏幕边缘，奶声奶气地催促：“说好要专心搞定的呢！怎么又在看视频啦，快回来啦～”
- **深夜疲劳**：宠物会打哈欠，揉揉眼睛，心疼地劝说：“已经写了三个小时了，小宠的眼皮都打架了，我们今天就到这里，赶紧休息好不好？”

### 3.3 隐私安全红线
- **本地优先**：所有屏幕和行为数据完全在本地计算和消耗，绝不上传云端，保障绝对的隐私安全。
- **脱敏提取**：仅提取非敏感的特征结构（例如“停留时长”、“应用类型属性”），屏蔽一切密码、个人身份及敏感聊天内容。

## 4. 智能网关架构与大模型对话 (Agent Gateway & LLM Interaction)

要让悬浮宠物的“智能”既强大又轻量，NeoCompanion 采用**“智能连接网关 + 面板适配”**的轻量化 Agent 架构。

我们不从零构建复杂的重度 Autonomous Agent，而是让 NeoCompanion 成为一个精致的**连接枢纽与控制台**，把复杂的任务分发给市面上强大的专业工具，并将执行状态优雅地收拢并展示在桌宠面板中。

```
                      +-------------------+
                      |   NeoCompanion    |
                      |   悬浮桌宠 / 网关  |
                      +-------------------+
                        /               \
                       /                 \  API 直连大模型
                      /                   \
            +------------------+     +--------------------+
            |    OpenClaw      |     |     LLM API        |
            | (负责复杂 GUI 执行) |     | (极简聊天/代码建议) |
            +------------------+     +--------------------+
```

### 4.1 内置基础原子 Agent 能力
本地直接提供简单、高频的单步读写与搜索能力，以便快速响应用户：
- **文件读写 (Read/Write)**：快速帮用户读取本地记录、保存备忘或更新待办文件。
- **网页搜索 (Search)**：在用户卡壳时，快速静默检索解决思路并总结。
- **剪贴板联动 (Clipboard)**：直接拾取用户复制的报错信息或文本。

### 4.2 OpenClaw 复杂任务网关 (Computer Use Gateway)
当用户给桌宠发送需要复杂跨应用操作、自主规划执行的长流程任务时（例如：“帮我把刚才整理的日记草稿发布到我的博客后台，并自动生成一篇总结推文发到社交平台”）：
- **网关角色**：NeoCompanion 本身不直接去模拟鼠标点击网页或跨软件流转，而是作为**智能连接网关**，直接对接并调用开源个人 AI 助手与 GUI 执行 Agent —— [OpenClaw](https://github.com/openclaw/openclaw)。
- **面板适配展示**：NeoCompanion 通过 API/WebSocket 订阅 OpenClaw 的执行进度、规划树和截图。桌宠会在桌面以“宠物正在后台施法”、“进度条加载”或小看板的形式，提供极其友好且不占屏幕的**面板适配展示**，让复杂 Agent 过程完全可视化、可控化。

### 4.3 极简 AI 对话与开发辅助面板 (Lightweight AI Chat & Coding Assistant)
为了确保桌宠的轻量与专注，我们不强塞复杂的 IDE 级编程控制面板，而是将其自然融入到桌宠的“极简 AI 对话框”中。
- **自然提问与闲聊**：用户在写代码或学习过程中遇到瓶颈、卡壳、或产生焦虑情绪时，可以直接在桌宠的聊天面板中倾诉或提问。
- **一键拾取与调试建议**：在核心的编程和报错排查场景中，用户可以直接使用“剪贴板自动抓取”或“一键填入报错”功能，向宠物发起提问。宠物将调用底层的通用大模型（如 DeepSeek/Claude）生成调优思路和标准的 Markdown 代码块。
- **快速复写与复制**：生成的代码和解决方案会清晰呈现在对话气泡中，支持一键复制代码。这既能有效帮助用户理清思路、打破卡壳，又保持了桌宠作为“桌面数字陪伴者”的轻量化本色。

## 5. 可自定义 Hook 状态跟踪与扩展机制 (Custom Hooks & Reminders)

为了给开发者和极客用户提供极致的自定义空间，NeoCompanion 提供了强大的 **Hook 连接机制**。

日常使用中，用户可以把 NeoCompanion 作为统一的**数字状态看板与拟人化提醒中心**，通过 Hook 连接任何第三方自定义程序。

```
+--------------------+                 Hook 推送 (HTTP/WS)
| 用户自定义程序/脚本  | ------------------------------------+
| (如编译脚本/监控脚本) |                                     |
+--------------------+                                     v
                                                  +-----------------+
                                                  |  NeoCompanion   |
                                                  |  桌宠拟人化提醒   |
                                                  +-----------------+
                                                  | "报告主子！刚才的 |
                                                  | 编译通过啦，真棒！"|
                                                  +-----------------+
```

### 5.1 本地 Hook 接口设计
- **简易 HTTP API / WebSocket 接收端**：程序在本地监听一个安全端口（例如 `localhost:10103`），提供标准的 JSON 接口，接收外部推送。
- **自定义状态定义**：外部推送 of JSON 结构可包含：
  - `status_type`: 状态类型（如：成功、警告、严重错误、日常通知）。
  - `message`: 具体消息文本。
  - `pet_action`: 期望宠物做出的指定动作/表情编号。
  - `tts_voice`: 是否开启语音反馈，以及指定的语音文案。

### 5.2 典型自定义 Hook 实例
1. **开发者编译与部署通知**：
   - 编写一段 `git push` 或代码构建后的 Hook 脚本。
   - 一旦编译成功，脚本 Hook 推送给桌宠。桌宠在屏幕上跳舞并用语音欢呼：“主子！服务器部署成功啦，完全没有报错，太牛了！”
   - 一旦构建失败，桌宠则抱着头露出哭泣的表情，语音提示：“啊呜，主子，编译又红了，快去看看报错日志吧，呜呜呜。”
2. **服务器与服务监控提醒**：
   - 监控本地或远程服务的健康状态。一旦服务挂掉，自动 Hook 触达桌宠，宠物做出紧张的动作发出语音警报。
3. **第三方智能家居与健康跟踪**：
   - 比如番茄钟久坐、饮水机提醒、社交媒体收到新私信，皆可通过轻量 Hook 直接让悬浮桌宠为您播报。

### 5.3 多通道极客 Hook 联动架构 (Advanced Multi-channel Hook & Permission Review)

为了将桌宠的“网关提醒”与“状态跟踪”体验做到极致，我们吸收并借鉴了优秀极客项目 `clawd-on-desk` 的多通道机制，设计了以下深度联动方案：

#### 5.3.1 配置文件扫描与自动打桩 (Auto-Patching & Detection)
- **零配置即用**：NeoCompanion 启动时，会自动扫描本地主流 AI 编程终端及 Agent 工具（如 OpenClaw 等）的配置文件路径（例如 `~/.openclaw/openclaw.json`）。
- **自动注册插件**：若检测到相应工具已初始化，系统会自动将 NeoCompanion 的 Hook 接口路径作为状态插件注入其中，无需用户繁琐地手动配置。当这些 Agent 工具运行时，便会自动向 NeoCompanion 实时广播其执行状态。

#### 5.3.2 浮动权限卡片与快捷审批 (Permission Bubble & Hotkeys)
- **免死等终端**：当底层的 GUI 执行器（如 OpenClaw）或本地自定义进程请求执行敏感指令（如修改核心系统配置、执行高危 bash 命令、覆盖关键文件）时，外部工具会通过 HTTP API 挂起当前流程并向 NeoCompanion 发送审批请求。
- **透明审批浮窗**：桌宠会直接在桌面弹出一个精美、半透明的“权限气泡审批卡片 (Permission Bubble)”，展示操作详情（操作类型、涉及路径、敏感度评分）。
- **极速热键反馈**：用户可以直接点击审批卡片中的 `Allow (同意)` 或 `Deny (拒绝)`。极客用户可以使用系统级全局热键：`Ctrl+Shift+Y` 批准，`Ctrl+Shift+N` 拒绝。
- **状态流控**：审批结果通过 HTTP/WebSocket 异步返回给挂起的执行进程，让终端交互实现从命令行到桌面有温度交互的跃升。

#### 5.3.3 微观状态映射引擎 (Event-to-Animation Mapping)
桌宠通过 Hook 捕获到外部 Agent 的微小执行状态时，能完美映射到宠物的细腻动画和动作中：
- **Thinking (思考中)**：Agent 正在梳理规划、调用模型时，桌宠做出托腮、思考、或转笔动作。
- **Working/Typing (工具执行中)**：Agent 正在跑工具、修改文件时，桌宠戴上眼镜在迷你键盘上工作。
- **Error (发生错误)**：任务失败或编译出错时，桌宠抱头哭泣或做出生气动作，并伴有心疼的语音安慰。
- **Subagent Juggling (多子智能体协作)**：当 Agent 启动多进程、多子智能体并行协作时，桌宠开始耍三个杂耍球，象征高效运转。

## 6. 阶段演进路线 (Roadmap)

### 6.1 Phase 1: 温暖的陪伴桌宠 (Warm Desk-Pet Base)
- **目标**：实现桌面悬浮宠物的 Live2D 动态展示、可爱的语音 TTS 反馈。
- **功能**：落地宠物番茄钟、宠物 Todo 小账本、日常天气播报和记录日记。
- **感知**：完成基本的窗口活跃度检测，驱动宠物对“专注/分心”做出语音和表情互动。

### 6.2 Phase 2: 自定义 Hook 与网关扩展 (Hooks & Gateway Integration)
- **目标**：打通可扩展的连接生态。
- **功能**：
  - 发布本地 Hook API，允许用户通过脚本向桌宠推送自定义状态和语音提醒。
  - **实现自动配置文件扫描注入**（优先支持 OpenClaw 的配置文件自动注册打桩）。
  - **落地“浮动权限审批气泡 (Permission Bubble)”机制**与全局系统热键操作（`Ctrl+Shift+Y` / `Ctrl+Shift+N`）。
  - 完善本地 AI 聊天面板的多模型自定义配置，提供一键极速答疑的开发辅助体验。
- **验证**：实现“外部 AI 执行敏感动作、代码编译成功/失败后，桌宠动态反馈和桌面一键审批”的极客闭环。

### 6.3 Phase 3: 智能 Agent 完整联接 (Full Agent Gateway Ecosystem)
- **目标**：全面作为个人工作流的智能控制面板。
- **功能**：
  - 深度打通 OpenClaw 的 GUI 执行网关，实现后台复杂 GUI 任务的代行与面板进度适配。
  - 扩展长久记忆系统，使宠物的声线、语气、建议策略随着长期的陪伴而愈发懂你，形成真正独一无二的“数字搭子”。
